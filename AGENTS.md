# AGENTS.md

## Проект
Пафосный, кинематографичный, полностью адаптивный React-лендинг для проекта миграции с Serious Engine 1.5 на Unreal Engine 5.7.

Цель текущего этапа: документация, выбор технологий, согласование UX/визуальных принципов, подготовка к быстрой итеративной разработке с визуальной проверкой.

## Ключевая идея дизайна
- Визуальный тон: cinematic + premium + military-tech (без шаблонного "фиолетового неона").
- Сильная типографика, крупные композиции, акцент на драматическом контрасте и глубине.
- Каждая секция должна быть как отдельный "кадр трейлера", но единый бренд-сценарий.

## Секции лендинга (MVP порядок)
1. Hero (видео фон, логотип, релиз-таймер, 3 CTA, правый блок-переключатель, левые соц-иконки 50x50)
2. Предрегистрация (анимированный счетчик, email, popup подтверждения c кодом + никнеймом)
3. Поддержка разработки (кнопка, крупная сумма донатов, описание наград за рубежи)
4. Новый мир (до/после со слайдером сравнения)
5. Роадмап (горизонтальный прогресс с отметками, подписями, стрелкой продолжения)
6. Медиа (скриншоты и видео)

## Рекомендуемый стек
- Framework: Next.js 15 (App Router) + React 19 + TypeScript
- Styling: Tailwind CSS 4 + CSS modules для сложных эффектов
- Animation: Framer Motion + GSAP (только для тяжелых скролл-сцен)
- UI primitives: shadcn/ui (минимально, без потери уникальности)
- Forms: React Hook Form + Zod
- Data fetching: TanStack Query (если появится API)
- Media: next/image, next/video pattern, adaptive sources
- State: Zustand (локальные UI-стейты)
- Tests: Playwright (e2e + визуальные smoke), Vitest + RTL (компоненты)
- Lint/format: ESLint + Prettier

## Почему такой стек
- Быстрый SSR/SEO и хорошая производительность для лендинга.
- Простая масштабируемость, если позже лендинг станет порталом.
- Удобный DX для регулярных правок дизайна и анимаций.

## Агентный workflow (обязательный)
1. Реализуем секцию/фичу небольшим инкрементом.
2. Сразу запускаем локальный просмотр и проверяем layout на брейкпоинтах.
3. Фиксим визуальные дефекты до перехода к следующей задаче.
4. Только потом двигаемся дальше.

## Визуальная проверка (Definition of Done на каждую секцию)
- Desktop: 1920x1080, 1440x900
- Tablet: 1024x1366, 834x1194
- Mobile: 430x932, 390x844, 360x800
- Проверяем:
  - не уезжают CTA/иконки/переключатели
  - нет наложения текста на критические зоны видео/изображений
  - интерактивные элементы доступны пальцем (>=44px hit area)
  - контраст и читаемость AA минимум

## Как "видеть сайт" в режиме агента
Чтобы агент сам быстро ловил косяки интерфейса:
- Запускаем dev сервер (`npm run dev`)
- Добавляем Playwright visual smoke по ключевым вьюпортам
- Делаем автоскриншоты каждой секции после изменений
- При регрессии: сразу правка + повторная проверка

Ограничение: агент не "смотрит" как человек напрямую в браузер, но может делать воспроизводимую визуальную проверку через e2e/скриншоты и править очень быстро.

## Художественный текст (тиснение/гравировка/премиум)
Рекомендуемый подход (без ущерба читабельности):
- Многослойный текст:
  - базовый цвет
  - `text-shadow` для глубины
  - тонкий внутренний свет/градиент через background-clip
  - аккуратный noise overlay
- Стили на display-заголовки, но body-текст оставляем максимально читаемым.
- 2 уровня декоративности:
  - Level 1 (везде): мягкий премиум-эффект
  - Level 2 (hero/названия): сильное тиснение

## Шрифтовая стратегия
- Display: выразительный акцентный шрифт (например, Cinzel / Orbitron / Bebas Neue как референсы)
- UI/Text: нейтральный современный шрифт для читаемости
- Не больше 2 семейств одновременно
- Обязательно fallback-цепочки

## Ограничения производительности
- Hero video: poster + preload strategy + ленивые источники
- Анимации только GPU-friendly (transform/opacity)
- Ограничить blur/backdrop-heavy эффекты на mobile
- Lighthouse Perf target: 85+ (mobile)
